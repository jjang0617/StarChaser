# StarChaser Phase 1 — 업무 분담 (조정안)

**기준 문서:** 스타체이서 MVP 간소화 기획서 v5.3  
**변경 사항:** **가상 밤하늘 뷰어** 담당을 **C → A(장성재)** 로 이동. 나머지 역할은 겹침을 줄이고 의존 관계를 명확히 재배치함.

**원칙:** 각 기능의 **FE + BE** 를 오너가 함께 책임진다(기획서 5장). 배포·인프라 고도화는 Phase 2 후순위.

---

## A · 장성재 (팀장)

**한 줄:** 백엔드 리드 · **Star-Index** · **인증** · **가상 밤하늘 뷰어(전담)** · 관측 기록·보정

### 팀장으로서

| 영역 | 내용 |
|------|------|
| **스프린트·일정** | 2주 단위 목표·우선순위 확정, 스코프 밀릴 때 **기능 축소·연기** 결정(기획서 6장: 일정 밀리면 커뮤니티 등 후순위 배제). |
| **품질 게이트** | 기획서 준수: **Star-Index 적중률·9~10변수 매핑**이 기준에 못 미치면 **신규 기능 Merge 보류** 여부 판단(§0.4, §7.4). |
| **크로스 체크** | **C**와 **Star-Index·달 고도·천문 데이터** 정합성 주기 점검(스프린트 리뷰 첫 안건 권장). **B**와 **지도·TOP5·앱 플로우** 통합 이슈 조율. |
| **보안·설정** | `.env`·API 키(기상청·에어코리아·KASI·Kakao 등) **관리 원칙** 수립, 저장소 비노출 확인(기획서 §9.3). |
| **리스크·에스컬레이션** | 외부 API 장애·캐시 Cold Start 등 **팀·베타 안내** 문구 합의, 블로커 시 우선순위 재배치. |

> 팀장이 **모든 코드를 다 쓰는 것**은 아니다. **Star-Index·인증·밤하늘 뷰어**는 A가 오너로 직접 구현하고, 위 표는 **병목·품질·협업**을 끝까지 책임지는 역할이다.

### 담당 기능

**Star-Index (10변수) + 실시간 보정 제보**

| 구분 | 내용 |
|------|------|
| **FE** | 홈 게이지, 제보 UI, Night Vision 연동 |
| **BE** | Cron·캐시, 보정 집계, `correction_score` 반영 |

**가상 밤하늘 뷰어**

| 구분 | 내용 |
|------|------|
| **FE** | Canvas/SVG 렌더링, 자이로·시간 UI, 별자리 오버레이 |
| **BE** | Hipparcos 등 별 데이터 API(또는 정적 번들), 천구 투영·시각·시간 계산, `SkyModule`·KASI와 일관성 |

**관측 기록·리포트**

| 구분 | 내용 |
|------|------|
| **FE** | 기록 폼, 성공/실패, 리포트 카드 |
| **BE** | Observations API, `weather_snapshot` 저장 |

**인증 (JWT·bcrypt·Rate limit)**

| 구분 | 내용 |
|------|------|
| **FE** | 로그인·회원가입 |
| **BE** | `AuthModule` 전반 |

### 스프린트 (제안)

**1–2주차**

- 가상 밤하늘 뷰어 MVP — 천구·별 렌더 1차, API 또는 정적 데이터 연동
- Star-Index 보정 API·제보 UI 초안

**3–4주차**

- 밤하늘 — 자이로·별자리 라벨 polish, 천문 계산 QA
- 관측 기록·리포트 완료, 보정 로직 안정화

## C · 지영재

**한 줄:** **DB·시딩** · **푸시 알림(이벤트 기반)** · **기본 촬영 가이드** · KASI/천문 데이터 검증 지원

### 담당 기능

**이벤트 기반 알림**

| 구분 | 내용 |
|------|------|
| **FE** | 알림 설정·수신 UI, 온보딩 토글과 동기 |
| **BE** | FCM·스케줄러(Star-Index 70+, 유성우·천체 이벤트, TOP5 갱신 알림), 고정 시각 일괄 발송 금지 |

**기본 촬영 가이드**

| 구분 | 내용 |
|------|------|
| **FE** | 정적 화면(기종별 ISO·셔터 등) |
| **BE** | 없음(API 불필요) 또는 마크다운/JSON 자산만 |

### 타 담당과 협업

- 밤하늘 뷰어 구현은 **A** 담당. **C**는 **KASI 달 위상·고도** 등 **교차 검증(Stellarium 등)**·기획서 7.4 **변수 매핑 점검**을 **A와 주기적으로 페어**한다.
- **명소 50개 시딩**의 `elevation_m`·좌표 정확도는 **C 주도**, 지도·검수는 **B**와 병행한다.

### 스프린트 (제안)

**1주차(29일까지)**

- DB 마이그레이션, 시딩 확대(목표 50개 중 가능한 만큼)
- FCM 토큰 저장 설계
- 알림 스케줄러 설계, 온보딩 알림 prefs 연동 착수

**3–4주차**

- 이벤트 조건별 푸시 실발송
- 촬영 가이드 콘텐츠·화면
- 알림 중복 방지·문구 검수
