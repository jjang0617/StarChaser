# Star-Index 태양 고도 보정 로직 수정 (PR-A)

> 대상 파일: `backend/src/star-index/sun-observation-score.util.ts`
> 관련 테스트: `backend/src/star-index/sun-observation-score.util.spec.ts`
> 정책 방향: StarChaser Star-Index는 "관측 실패를 줄이는 보수적 점수"로 운영한다.

## 1. 배경

Cursor·Claude 교차 검증과 팀장 확인 결과, `applySunAltitudeToStarIndexScore()`의
박명(천문박명~일출) 구간 보정이 **버그**로 판정되었다.

별 관측은 태양이 지평선 아래로 더 깊이 내려갈수록(하늘이 어두워질수록) 유리하다.
따라서 태양 고도가 **높아질수록** 보정 multiplier는 **낮아져야(단조 감소)** 한다.

## 2. 변경 전 문제

기존 구간식(`baseScore`에 곱하는 multiplier 기준):

| 구간 | 기존 식 | 거동 | 문제 |
|---|---|---|---|
| -18° ~ -12° | `0.75 + t*0.25` | 0.75 → 1.00 (증가) | 역방향 |
| -12° ~ -6° | `0.43 + t*0.32` | 0.43 → 0.75 (증가) | 역방향 |
| -6° ~ 0° | `0.45 - t*0.30` | 0.45 → 0.15 (감소) | 정상 |

추가로 **-12° 경계에서 multiplier가 약 1.00 → 0.43으로 급락**하는 불연속이 존재했다
(동일 base 점수가 경계를 지나는 순간 약 57% 급감).

형제 함수 `sunAltitudeToObservationScore()`(UI 표시용 `daylight_observation_score` 생성)도
동일한 역방향·불연속 패턴을 갖고 있었다.

## 3. 변경 내용

### 3-1. `applySunAltitudeToStarIndexScore()`
박명 구간 전체(-18°~0°)가 단조 감소하도록, 그리고 경계에서 연속이 되도록 재정의했다.

승인된 앵커:

| 태양 고도 | multiplier |
|---:|---:|
| ≤ -18° | 1.00 |
| -12° | 0.75 |
| -6° | 0.45 |
| 0° | 0.15 |
| > 0° | 기존 낮 시간 강한 제한 정책 유지 |

수정된 구간식(piecewise-linear, 경계 양쪽 값 일치):

| 구간 | 수정 식 | 거동 |
|---|---|---|
| -18° ~ -12° | `1.00 - t*0.25` | 1.00 → 0.75 |
| -12° ~ -6° | `0.75 - t*0.30` | 0.75 → 0.45 |
| -6° ~ 0° | `0.45 - t*0.30` (기존 유지) | 0.45 → 0.15 |

`t`는 각 구간 내 0→1 정규화 값. `sunAltDeg < -18`(천문박명 이하)은 base 그대로,
`sunAltDeg >= 0`(낮)은 기존 강한 제한 정책을 유지한다.

### 3-2. `sunAltitudeToObservationScore()`
표시용 점수를 본점수 multiplier와 동일하게 맞추기 위해
`applySunAltitudeToStarIndexScore(100, sunAltDeg)`에 위임하도록 단순화했다.
이로써 표시 점수(완전한 밤=100)와 실제 점수의 박명대 거동이 항상 일치한다.

## 4. 적용 이유 (왜 이렇게 했는가)

- **방향성**: 천문학적으로 태양 고도가 높을수록 천광(하늘 밝기)이 증가 → 관측 불리.
  multiplier 단조 감소가 물리적으로 옳다.
- **연속성**: 경계에서의 급락은 같은 환경인데 1° 차이로 점수가 크게 튀는 비정상 현상을 만든다.
  앵커 사이를 선형 보간하고 경계 양쪽 식의 값을 맞춰 점프를 제거했다.
- **표시·본점수 일관성**: 표시용 함수가 본점수와 다른 곡선을 쓰면 사용자 혼란을 유발하므로 통합했다.
- **보수적 정책 유지**: 낮 시간(>0°)의 강한 상한 제한은 그대로 두어 "주간 측정불가" 정책을 유지했다.

## 5. 테스트

`sun-observation-score.util.spec.ts`에 추가:

- 박명 구간 multiplier 단조 감소 (역방향 회귀 방지)
- 경계 앵커 값 검증 (-18°=100, -12°=75, -6°=45, 0°=14~15)
- 경계(-18/-12/-6/0)에서 점프 없이 연속(허용 오차 ≤ 2점)
- 표시용 관측 점수 단조 감소

기존 테스트(낮 시간 점수 억제, 천문박명 이하 base 유지, 박명 부분 감점)는 그대로 통과한다.

## 6. 예상 점수 변화

- 박명대(-18°~-6°)에서 기존보다 점수가 **상향·매끄럽게** 산출된다.
- 특히 -12° 직전의 비정상 급락이 사라진다.
- 천문박명 이하(< -18°)와 낮(> 0°)은 사실상 변화 없음.
