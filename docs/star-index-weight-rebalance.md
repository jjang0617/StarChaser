# Star-Index 가중치 재조정 및 보정 정책 변경 (PR-B)

> 대상 파일: `backend/src/star-index/star-index-scoring.util.ts`
> 관련 테스트: `backend/src/star-index/star-index-scoring.util.spec.ts`
> 정책 방향: Star-Index는 "관측 실패를 줄이는 보수적 점수"로 운영한다.
> 선행/분리 PR: 태양 고도 보정 버그 수정은 별도 PR(`fix/star-index-sun-altitude`)에서 진행.

## 1. 배경

Cursor·Claude 교차 검증 + 팀장 의사결정 결과를 반영한다. 핵심 결정:

1. Star-Index는 실패를 줄이는 보수적 점수로 간다.
2. 서비스 타깃은 은하수/딥스카이가 아니라 **밝은 별/별자리 육안 관측**이다.
3. PM2.5는 현재보다 더 강하게 감점한다.
4. 고도 변수는 유지하되 가중치를 올린다.
5. 기온은 본점수에서 중립 처리한다.
6. 운량/강수는 가중치 + gate 이중 반영을 유지한다.
7. 사용자 제보는 바로 점수에 자동 반영하지 않고 로그/데이터로만 수집한다.

## 2. 가중치 재분배

`correction`(0.074)을 점수에서 제거(가중치 0)하여 회수하고, 과소·실효성 부족으로
지적된 항목에 재분배했다. **합계는 1.0을 유지**한다.

| 변수 | 변경 전 | 변경 후 | 증감 | 근거 |
|---|---:|---:|---:|---|
| cloud | 0.273 | 0.273 | - | 실패직결 핵심, 유지(+gate 이중 반영) |
| light | 0.236 | 0.236 | - | 유지 |
| moon | 0.145 | 0.145 | - | 밝은 별/별자리 중심 → 큰 상향 보류(P6) |
| **pm25** | 0.100 | **0.130** | +0.030 | 겨울철 미세먼지 실패 반영 강화 |
| precip | 0.073 | 0.073 | - | 유지(+gate 이중 반영) |
| wind | 0.036 | 0.036 | - | 유지 |
| **elevation** | 0.027 | **0.040** | +0.013 | 변별력 보강(보조 변수라 소폭) |
| **humidity** | 0.018 | **0.038** | +0.020 | 과소평가된 투명도 계열 보강 |
| **visibility** | 0.018 | **0.029** | +0.011 | 투명도 보강(미수신 시 cloud로 흡수) |
| **correction** | 0.074 | **0.000** | −0.074 | 자동 점수 반영 비활성화 |
| **합계** | **1.000** | **1.000** | 0 | |

회수량 0.074 = pm25(+0.030) + elevation(+0.013) + humidity(+0.020) + visibility(+0.011).

### 잉여 가중치 배분처 판단 (humidity/visibility 선택 이유)
- cloud/precip은 이미 가중치 + gate로 충분히 강함 → 추가 집중 시 변별력 저하 → 제외.
- light/moon은 정책상 유지/보류.
- 두 AI가 공통으로 과소 지적한 **투명도 계열(습도·시정)** 보강이 균형상 가장 타당.
- humidity는 PM2.5 연무 감점과 일부 겹치므로(`calcHumidityHazeScore`) 과도 상향은 지양.
- visibility는 KMA 시정 미수신 시 가중치가 cloud로 흡수되므로 humidity보다 작게 배분.

## 3. 사용자 제보(correction) 처리 변경

- `correction` 가중치를 **0**으로 두어 실시간 점수 자동 반영을 비활성화했다.
- 제보 저장·집계 경로(`CorrectionsService`, 제보 엔티티)는 **변경하지 않고 유지**한다.
  → 제보는 계속 수집되며 운영 분석 데이터로 활용한다.
- 함수 시그니처·`correction_score` 산출/스냅샷 저장은 그대로 유지 → 추후 데이터가
  쌓이면 사람이 검토 후 수동으로 가중치/지역 보정에 반영 가능.

## 4. 정책 문서화 (코드 주석 반영)

- **운량/강수 이중 반영**: `cloudTransmittanceGate`, `precipitationGate`에 "의도된 보수
  정책"임을 주석으로 명시. 흐림·강수 조건에서 점수가 강하게 낮아지는 것은 의도다.
- **기온 중립**: `calcTemperatureNeutralScore`에 "기온은 체감·안전 요소에 가까우므로
  MVP 본점수에는 반영하지 않고 중립(100) 처리, 스키마 호환 위해 키만 유지" 명시.

## 5. 테스트

`star-index-scoring.util.spec.ts`에 추가:

- blend 가중치 합계 1.0 검증 (모든 하위 점수 100 + gate 무손실 → 100)
- correction_score(0/50/100)가 달라도 최종 점수 동일 (자동 반영 비활성화 회귀 방지)
- PM2.5 매우나쁨이 보통보다 총점을 더 크게 끌어내림

## 6. 예상 점수 변화

- `correction_score`가 기본 100(최댓값)으로 baseline +7.4점을 주던 효과가 사라지고
  과소 변수로 재분배되므로, 일반적인 점수가 **소폭 하향**된다(보수적 정책과 부합).
- 미세먼지 나쁜 날·저지대 점수가 추가로 하향, 고지대·청정일은 상대적으로 부각.
- 운량/강수/달/광공해 등 핵심 요인의 거동은 유지.
